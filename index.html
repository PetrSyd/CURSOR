Шпаргалка

  • git config --global user.name blabla :Задати ім’я
  • git config --global user.email blabal@bla :Задати e-mail
  • git init :Ініціалізувати порожній локальний Git-репозиторій
  • git status :Огляд поточного стану вашого сховища
  • git add :Додати всі файли в індекс
  • git commit :Покласти зміни в локальний репозиторій
  • git log :перераховує комміти, зроблені в репозиторії у зворотному до хронологічного порядку - останні комміти знаходяться вгорі
  • git checkout :Перемикання між різними версіями цільового об'єкта
  • git branch :Показати лише локальні гілки
  • git push :Покласти зміни з локального у віддалений репозиторій
  • git pull :Оновити дані (забрати зміни) з віддаленого репозиторію по поточній гілці
  • git merge :Поєднує кілька послідовностей коммітів у спільну історію
  • git clone :Клонувати репозиторій (наприклад по HTTP https://github.com/{vendor_name}/{project_name}.git)

  • .gitignore файл :це файл який використовується для виключення файлів та каталогів зі зберігання в репозиторії. Він містить список шаблонів імен файлів та директорій, які Git повинен ігнорувати при виконанні команди "git add"

  • git revert :Створює новий коміт який відміняє зміни зроблені в попередньому коміті. Не видаляє попередній коміт, а створює новий коміт з змінами щоб відмінити попередні зміни
  • git reset :Змінює стан гілки на попередній
  • git rebase :Дозволяє змінити історію комітів, зберігаючи при цьому цілісність репозиторію
  • git cherry-pick :Дозволяє переносити окремі зміни з однієї гілки в іншу, не переносячи всі зміни з гілки
  • git stash :Зберігає непідтверджені зміни в окремому сховищі, ви можете вносити зміни, перемикатися між гілками і виконувати інші операції Git. Відкладені зміни можна буде застосувати пізніше. Відкладені зміни зберігаються у локальному репозиторії Git і не передаються на сервер
